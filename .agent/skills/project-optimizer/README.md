# 项目优化大师 (Project Optimizer) - 目录说明

## 📁 文件结构

```
project-optimizer/
├── SKILL.md                           # 核心技能文档
├── README.md                          # 本文件
├── interactive-diagnostic.md          # 交互式诊断问卷
├── quick-reference.md                 # 快速参考指南
└── examples/
    ├── growth-analysis-framework.md   # 商业增长分析框架
```

---

## 📖 文件说明

### 核心文档

#### `SKILL.md`
**主技能文档**，定义了 5 大分析维度：
1. 商业与增长（获客、留存、变现）
2. 功能补全与延伸（上下游场景、竞品降维）
3. 用户体验（视觉、交互、反馈）
4. 性能优化（速度、响应时间）
5. 技术债务（安全、质量）

**使用场景**: 了解完整分析框架和使用方式

---

### 引导工具

#### `interactive-diagnostic.md`
**交互式问卷**，通过 9 个阶段的提问引导用户提供项目信息：
- 项目定位与技术栈
- 痛点挖掘（视觉/性能/功能）
- 增长与商业分析
- 优先级排序

**使用场景**: 当你不确定如何描述项目时，按问卷逐步填写

#### `quick-reference.md`
**快速检查清单**，整合了性能、UX、技术债务的关键检查点。

**使用场景**: 
- 快速自查项目现状
- 寻找优化切入点

---

### 扩展示例

#### `examples/growth-analysis-framework.md`
**商业增长详细框架**，深入讲解：
- 获客转化策略
- Aha Moment 识别
- 留存钩子设计  
- 分享裂变机制
- 上下游延伸思维

**使用场景**: 聚焦商业增长和用户留存优化

#### `examples/sample-optimization-report.md`
**实际案例**：SaaS Marketplace 项目的完整优化清单。

**使用场景**: 参考优化报告的格式和思路

## 🚀 快速开始

### 方式 1: 交互式诊断（推荐新手）
```
请以项目优化大师身份，通过提问了解我的项目，并产出优化清单。
```

### 方式 2: 直接分析（已有明确信息）
```
项目背景：[一句话描述]
技术栈：[前后端]
困扰：
1. [具体问题 + 数据]
2. [用户反馈]

请基于以上信息产出优化清单。
```

### 方式 3: 专项分析
```
聚焦[商业增长/用户体验/性能优化]，分析 [项目名] 的优化机会。
```

## 💡 核心理念

- **商业优先**: 优化服务于业务目标
- **用户视角**: 从真实痛点出发
- **ROI 导向**: 快赢项目优先
- **数据说话**: 用数据验证和预估
- **竞品对标**: 知己知彼
