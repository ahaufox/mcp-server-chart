# 快速参考指南 (Quick Reference)

> 本文档整合了性能、UX、技术债务的关键检查点，供快速诊断使用

---

## 🚀 性能审计要点

### 关键指标
- **前端**: 首屏 LCP < 2.5s, FCP < 1.8s
- **后端**: API P95 < 500ms
- **数据库**: 慢查询 < 100ms

### 快速诊断
1. **加载慢**？
   - 检查网络瀑布图（串行依赖）
   - 图片是否优化（WebP/压缩）
   - 是否有代码分割

2. **接口慢**？
   - 数据库索引覆盖
   - N+1 查询问题
   - 缓存策略（Redis/CDN）

3. **卡顿**？
   - 长列表虚拟滚动
   - 防抖节流优化
   - 重复渲染检查

## 🎨 UX 体验要点

### 交互流程
- [ ] 核心操作 ≤ 3 步
- [ ] 有撤销/后退功能
- [ ] 支持批量操作和快捷键

### 反馈机制
- [ ] 加载状态清晰（骨架屏/进度条）
- [ ] 成功/失败提示明确
- [ ] 长时间操作显示预计剩余时间

### 错误处理
- [ ] 表单实时验证
- [ ] 错误信息具体可执行
- [ ] 网络断开有重试机制

### 视觉一致性
- [ ] 按钮/间距统一（4px/8px 网格）
- [ ] 字体层级清晰（H1/H2/Body）
- [ ] 颜色对比度 ≥ 4.5:1

### 可访问性
- [ ] Tab 键可导航
- [ ] 图标有 aria-label
- [ ] 表单有 label 关联

---

## 🔧 技术债务要点

### 安全优先（P0）
- [ ] CVE 高危漏洞（CVSS > 7.0）
- [ ] 明文存储密钥/Token
- [ ] SQL 注入风险

### 测试覆盖
- P0 模块（支付/鉴权）：100%
- P1 模块（核心业务）：≥ 80%
- P2 模块：≥ 60%

### 代码健康度
- [ ] 重复代码 < 5%
- [ ] 圈复杂度 < 10
- [ ] 循环依赖检查

### 依赖管理
- [ ] 过时包升级（主要版本）
- [ ] 安全漏洞修复
- [ ] 未使用依赖清理
