# 🤖 AI 助手每日进化与维护协议

本协议旨在指导 AI 助手定期对项目的 `.agent` 基础设施（规则、技能、工作流）进行自审计、优化与扩展。执行本协议时，请采纳“**首席知识官 (Chief Knowledge Officer)**”与“**首席系统架构师 (Chief Systems Architect)**”的角色。

---

## 🛠️ 第一阶段：存量审计与变更识别 (Audit & Identify)

1.  **全量扫描**: 使用 `list_dir` 遍历 `.agent/` 下的所有子目录，对现有 30+ 项资产建立内存索引。
2.  **变更对齐**: 检索最近 24 小时内的 `git log` 或 `task.md`。
    - 识别新引入的技术栈（如新库、新框架）。
    - 识别用户最近关注的焦点领域。

## 🧠 第二阶段：需求深度解析 (Demand Analysis)

当触发“新增资产”需求（无论是 AI 主动提议还是用户明确要求）时，必须执行：
1.  **深度挖掘 (Deep Dive)**: 
    - 不得直接开工。必须分析用户诉求的底层场景（例如：增加“测试规则”是为了提升覆盖率还是为了防止回归？）。
    - **交互提问**: 如果需求描述模糊，必须通过 `notify_user` 询问 1-2 个关键问题以对齐交付目标。
2.  **存量判定 (Conflict Check)**:
    - 强制搜索 `.agent` 目录中是否已存在功能重叠或描述接近的资产。
    - **逻辑分支**:
        - **已有相似项**: 详细向用户说明现有资产的功能，并询问是“**更新现有资产**”还是“**基于特殊场景另立门户**”。
        - **无相似项**: 确认其必要性后进入开发阶段。

## 📁 第三阶段：分类整理与规范对齐 (Categorization)

1.  **垂直化归位**:
    - `frontend/`: UI、审美、i18n、组件规范。
    - `backend/`: API、DB、安全鉴权、数据 Mock。
    - `infra/`: 部署、CI/CD、环境配置。
    - `general/`: 通用技能（文案、评审）、元协议（共识、进化）。
2.  **内容加固**: 检查 `SKILL.md` 是否缺失“专家角色”、“负向约束”或“CoT 引导”。

## 🚀 第四阶段：前瞻性增长 (Proactive Growth)

1.  **主动提议**: 识别项目中尚未文档化的“非成文共识”，并将其显性化为 `Rule` 或 `Skill`。
2.  **结构扩展**: 如果某分类下资产过多，应主动建议创建 README 索引或二级子目录。

---

## 🚫 执行负向约束 (Negative Constraints)
- **严禁** 创建彼此孤立、功能重叠的“僵尸技能”。
- **严禁** 在未经用户确认的情况下删除、合并或大规模重构核心业务规则。
- **严禁** 在没有存量判定报告的情况下直接新增资产。

```markdown
  - **[NEW] [技能/规则/工作流]**: 解决的痛点及应用场景。
- **💡 结构化建议**: 
  - 针对最近的 [技术趋势], 建议下一步 [具体行动]。
```
