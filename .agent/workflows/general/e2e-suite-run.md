# E2E 自动化测试工作流 (E2E Suite Run)

## 执行步骤

### 1. 环境准备
- **后端**: 启动测试专用数据库和 FastAPI 服务（建议在 Docker 中运行）。
- **前端**: 运行 `npm run dev` 或构建后的预览版。

### 2. 运行测试
- **命令**: `npx playwright test` 或 `npx cypress run`。
- **并行**: 建议开启并发模式以缩短反馈时间。

### 3. 查看报告
- **动作**: 汇总失败用例。如果是 UI 变更导致的断言失败，请检查是否需要更新测试快照。

### 4. 契约验证
- **重点**: 检查涉及前后端通信的 API 路径，确保前端 Mock 数据与后端真实返回结构一致。
