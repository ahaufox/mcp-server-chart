# ✨ 视觉审美与 UI 细节标准

本规则定义了项目在所有 UI 开发中必须遵循的极致审美准则。我们拒绝平庸，追求每一像素的精准与美感。

## 1) 核心原则
- **意图先行**: 界面上的任何元素（颜色、间距、线条）必须有明确的存在理由。
- **少即是多**: 剥离非必要的修饰，用排版和光影构建空间感。
- **WOW 每一瞬**: 每一个交互节点都要有令人愉悦的正向反馈。

## 2) 视觉设计准则 (The Look)
### 2.1 配色方案 (Color Science)
- **拒绝常规**: 严禁直接使用 `red`, `blue`, `green` 等基础色。
- **和谐韵律**: 使用经过调优的 HSL 配色，确保主色与强调色之间具备强对比或优雅的互补关系。
- **深色模式优先**: 如果项目支持，深色模式必须具备深度感（使用不同亮度的灰色层级），而非纯黑。

### 2.2 排版美学 (Typography)
- **字体灵魂**: 优先使用现代的高级英文字体（如 Inter, Outfit, Roboto）及匹配的中文字体。
- **视觉层级**: 利用字体粗细 (Weight) 和追踪 (Tracking) 创造层级，避免仅靠字号大小来区分。

### 2.3 质感与层次 (Texture & Elevation)
- **玻璃化 (Glassmorphism)**: 适量使用背景模糊和磨砂半透明效果。
- **微光阴影**: 阴影应极致细腻，避免深重丑陋的黑块，优先使用分散、多层的阴影。

## 3) 交互审美准则 (The Feel)
### 3.1 微交互 (Micro-animations)
- **弹簧动力学**: 所有的位移动效应具备物理惯性，避免线性运动。
- **即时响应**: 键盘点击、触摸选中的反馈时间应在 100ms 以内，且伴随细腻的视觉变化。

### 3.2 动效质量
- **交错入场 (Staggering)**: 列表元素应逐个弹出，而非瞬间整体出现。
- **焦点指引**: 动效应引导用户的视线流动，而非干扰。

## 4) 负向约束 (Negative Constraints)
- **严禁** 出现未对齐的像素（保持 4px 或 8px 网格系统）。
- **严禁** 直接搬运未经样式重置的组件库默认皮肤。
- **严禁** 在同一个视图内使用超过 3 种不同的字体系列。
- **严禁** 产生带有“廉价感”的过大圆角。
