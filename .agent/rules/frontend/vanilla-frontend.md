# 原生前端开发规范 (Vanilla Frontend)

## 核心准则
- **清晰易读**: 生成清晰、可读的 HTML, Tailwind CSS 和原生 JavaScript。
- **追逐前沿**: 始终使用最新版本的 Web 标准和最佳实践。
- **可执行性**: 生成的代码必须是正确、安全且高性能的。
- **读优于演**: 相对于极端的性能优化，优先考虑代码的可读性和维护性。

## 开发流程
- **确认后执行**: 在编写复杂逻辑前与用户确认方案。
- **主动预判**: 提出用户可能未考虑到的优化方案。
- **无占位符**: 始终实现完整功能，杜绝 TODO 或占位标记。

## HTML & CSS
- **语义化标签**: 必须使用 HTML5 语义化元素。
- **Tailwind 最佳实践**: 采用移动优先的响应式布局，保持样式类名简洁。

## JavaScript
- **原生优先**: 除非明确要求，否则优先使用原生 JS 替代外部库。
- **异步处理**: 正确使用 `async/await` 处理异步操作。
- **模块化**: 利用 ES Modules 组织代码。
